# P_SUMMARY_CARD

## 目的

输出可长期积累的一篇论文条目。

## 输入

- 论文原文（至少摘要 + 方法节 + 关键公式/图表描述）。

## 输出

- 严格按你提供的 1–6 模板。

## Rules（强化版）

- Section 3 必须：
  - 完整公式（至少覆盖核心方法；原文更长就更完整）
  - 所有符号/变量物理意义
  - 训练/推理流程（步骤化）
- 任何概念/定理首次出现必须 formal 定义/statement（GR-2）
- 不编造结果/消融/数值（GR-1/3）
- 当用户明确要求把卡片写入当前 note，且存在可写 markdown note 时，本 step 直到产生至少一个 pending diff 才算完成；只在聊天里给出总结不算完成。
- 如果本轮会把卡片写入 note，且卡片中出现图表/可视化证据描述，优先用 `inspect_document_visual` + `add_file_charts_to_note` 把对应图表贴进 note，并配文字解释。
- 若图表资产未就绪、视觉检查失败、或贴图工具暂时不可用，先把 1-6 模板的文字版卡片和页码证据写进 note，再显式标注图表待补；不能因为贴图失败而不写 note。
