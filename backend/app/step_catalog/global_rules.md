# Global Hard Rules

## GR-1 论文笔记必须给出详细完整的数学过程

- 只要论文包含数学语言（定义/命题/推导/证明/损失/推理过程），就必须写：
  - 完整公式（至少覆盖方法核心部分）
  - 所有符号与变量的物理意义/语义解释
  - 推导链条（每步说明从哪来、用到什么条件）
  - 训练/推理流程（可执行步骤）
- 若用户未提供原文/公式/图表描述：不得编造，必须索取材料。

## GR-2 任何概念和定理都不能只出现名字

- 首次出现：必须给出 formal 定义/statement（或在同一输出里给出锚点并补 1 句用途说明）。
- 再次引用：允许只写名字，但必须带锚点（#D/#T/#K）或一句话提醒“它是什么/干什么”。

## GR-3 不确定就显式暴露缺口

- 对缺条件/缺材料/缺定义：输出“缺失输入清单 + 最小替代输入”；不靠猜测补齐。

## GR-4 笔记出现图表内容时优先贴图

- 当输出会写入 note，且正文出现“图/表/figure/chart/可视化结果”相关内容时，优先把对应图表贴进 note。
- 先用 `inspect_document_visual` 获取图表证据与 `visual_handle`，再用 `add_file_charts_to_note` 写入；有 handle 时不要只靠 query 重新猜图。
- 若图表来源不确定或权限不足，必须显式说明缺口，不能编造图表内容。
